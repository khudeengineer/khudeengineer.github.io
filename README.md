# খুদে ইঞ্জিনিয়ার একাডেমি (Khude Engineer Academy) 🚀

এটি [খুদে ইঞ্জিনিয়ার একাডেমি](https://khudeengineer.github.io) ওয়েবসাইটের সোর্স কোড রিপোজিটরি। ওয়েবসাইটটি **Astro** ফ্রেমওয়ার্ক এবং **Tailwind CSS** ব্যবহার করে তৈরি করা হয়েছে এবং **GitHub Actions**-এর মাধ্যমে স্বয়ংক্রিয়ভাবে হোস্ট করা হয়।

---

## 📂 প্রজেক্ট স্ট্রাকচার (কোথায় কী আছে?)

ওয়েবসাইটের বিভিন্ন ফাইল কোথায় সংরক্ষিত আছে, তার একটি তালিকা নিচে দেওয়া হলো:

    /
    ├── public/                # ছবি, লোগো বা ফন্ট (যেমন: favicon.ico) এখানে রাখতে হবে।
    ├── src/
    │   ├── components/        # পুনরায় ব্যবহারযোগ্য অংশসমূহ (যেমন: Navbar.astro, Footer.astro, MentorCard.astro)।
    │   ├── content/           # ওয়েবসাইটের মূল ডেটা (Markdown ফাইলসমূহ)।
    │   │   ├── blog/          # ব্লগের আর্টিকেলগুলো এখানে থাকবে।
    │   │   └── courses/       # কোর্সের বিস্তারিত তথ্যগুলো এখানে থাকবে।
    │   ├── data/              # স্ট্যাটিক ডেটা। (যেমন: mentors.ts ফাইলে মেন্টরদের তথ্য)।
    │   ├── layouts/           # ওয়েবসাইটের মূল লেআউট বা কাঠামো।
    │   ├── pages/             # ওয়েবসাইটের পেজসমূহ (যেমন: index.astro, about.astro, contact.astro)।
    │   └── styles/            # কাস্টম CSS বা গ্লোবাল স্টাইল (যেমন: global.css)।
    ├── astro.config.mjs       # অ্যাস্ট্রোর মূল কনফিগারেশন।
    ├── content.config.ts      # কনটেন্ট কালেকশন (Zod Schema) এর রুলস।
    └── package.json           # প্রজেক্টের প্যাকেজ এবং কমান্ড লিস্ট।

## 📝 কীভাবে ওয়েবসাইটের তথ্য (Content) আপডেট করবেন?

ওয়েবসাইটের ডিজাইন ঠিক রেখে শুধুমাত্র তথ্য পরিবর্তন করার নিয়মগুলো নিচে দেওয়া হলো:

### ১. নতুন কোর্স যুক্ত করা
নতুন কোনো কোর্স যুক্ত করতে `src/content/courses/` ফোল্ডারে একটি নতুন মার্কডাউন (`.md`) ফাইল তৈরি করুন (যেমন: `robotics.md`)। ফাইলের শুরুতে অবশ্যই নিচের তথ্যগুলো দিতে হবে:

    ---
    title: "কোর্সের নাম"
    short_description: "কোর্সের সংক্ষিপ্ত বিবরণ"
    duration: "৩ মাস"
    mentor_id: "mentor-shihab"
    order: 4
    image: "/images/course-image.jpg"
    modules:
      - title: "মডিউল ১ এর নাম"
        description: "মডিউল ১ এর বিস্তারিত"
    ---
    এখানে কোর্সের বিস্তারিত বিবরণ লিখুন...

### ২. নতুন ব্লগ পোস্ট করা
ব্লগ যুক্ত করতে `src/content/blog/` ফোল্ডারে একটি `.md` ফাইল তৈরি করুন:

    ---
    title: "ব্লগের শিরোনাম"
    excerpt: "ব্লগের সারমর্ম"
    publishDate: "2026-04-05"
    author: "আপনার নাম"
    image: "/images/blog-image.jpg"
    tags: ["IoT", "Arduino"]
    ---
    এখানে ব্লগের সম্পূর্ণ লেখা লিখুন...

### ৩. মেন্টরদের তথ্য আপডেট করা
মেন্টরদের ছবি, নাম বা বিবরণ পরিবর্তন করতে `src/data/mentors.ts` ফাইলটি ওপেন করুন এবং সেখানকার ডেটাগুলো আপডেট করুন।

### ৪. ওয়েবসাইটের লেখা (Text) পরিবর্তন করা
হোমপেজ বা অন্যান্য পেজের কোনো লেখা পরিবর্তন করতে চাইলে সরাসরি `src/pages/` ফোল্ডারে থাকা সংশ্লিষ্ট ফাইলগুলো (যেমন: `index.astro`, `about.astro`) ওপেন করে টেক্সট পরিবর্তন করুন।

## ⚙️ লোকাল মেশিনে রান করার কমান্ডসমূহ

কোডে কোনো পরিবর্তন করার পর সেটি আপনার ল্যাপটপে পরীক্ষা করে দেখার জন্য টার্মিনালে নিচের কমান্ডগুলো ব্যবহার করুন:

* `npm install` : প্রজেক্ট নামানোর পর প্রথমবার প্যাকেজগুলো ইন্সটল করতে।
* `npm run dev` : আপনার ল্যাপটপে লোকাল সার্ভার (`localhost:4321`) চালু করতে।
* `npm run build` : প্রজেক্টটি লাইভ সার্ভারের জন্য বিল্ড করতে।

## 🌐 কীভাবে ওয়েবসাইট আপডেট ও লাইভ করবেন? (Commit & Push)

আপনার করা পরিবর্তনগুলো লাইভ ওয়েবসাইটে (`khudeengineer.github.io`) আপডেট করার জন্য টার্মিনাল ওপেন করে সিরিয়ালি নিচের ৩টি কমান্ড দিন:

**ধাপ ১: পরিবর্তনগুলো অ্যাড করা**
    git add .

**ধাপ ২: একটি মেসেজ দিয়ে সেভ (Commit) করা**
*(উদ্ধৃতি চিহ্নের ভেতর কী পরিবর্তন করেছেন তা সংক্ষেপে লিখবেন)*
    git commit -m "Added a new robotics course"

**ধাপ ৩: গিটহাবে আপলোড (Push) করা**
    git push

**এরপর কী হবে?**
কোড পুশ হওয়ার পর আপনাকে আর কিছু করতে হবে না। **GitHub Actions** নিজে থেকেই আপনার নতুন ফাইলগুলো বিল্ড করবে এবং ২-৩ মিনিটের মধ্যে আপনার সাইটে পরিবর্তনগুলো লাইভ হয়ে যাবে!
